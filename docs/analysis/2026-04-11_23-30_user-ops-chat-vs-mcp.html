Bottom line
Of 556 external user operations in 48 hours, only 16% (87 ops) came from the internal chat — and chat writes were almost zero (20 writes, mostly one user enabling keywords). The real action happens through MCP clients: 84% of operations and 92% of all writes. 2 of 7 returnable users came back (29% D1), both business-domain MCP-heavy users.

So what
  1. 1 Chat is a discovery/onboarding funnel, not an action tool — 77% reads, and the 2 chat-only users (webrakes, alexandr via chat) stayed read-only. Optimize chat to nudge users toward their first write action: after 3+ reads, suggest a specific write (e.g., 'I found 5 irrelevant search terms — want me to add them as negatives?').
  2. 2 The chat-to-MCP handoff is where activation happens — 3 users explored via chat then shifted to MCP for bulk work. Make this transition smoother: after chat exploration, offer a 'copy MCP setup command' or 'continue in Claude Desktop' CTA.
  3. 3 Keyword management (add/pause negatives) is entirely an MCP workflow — 0 negative keywords added via chat. Consider making the chat agent more proactive about suggesting negative keywords after showing search term reports.
  4. 4 Both returning users (tim, m.sanchez) are MCP-only — no returner used chat. MCP users who write are the retention core. Focus retention efforts on getting MCP users to their first write, not on chat engagement.
